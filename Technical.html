为什么需要它

主流 GraphRAG 为开放网络设计 ——
但文学不一样。

当你问"这个角色相信什么"，答案必须只来自文本本身。每一条推断必须 引用原文。图谱要能处理 多角色视角、伏笔回收、反事实续写。

LoreGraph 把一部小说、剧本或电影分镜本，自动抽取成一个可查询的知识图谱 —— 每一条主张都锚定在原文的字面片段上 —— 并通过命令行和交互式 Web UI 暴露出来。

以严格的 证据片段字面匹配率 ≥ 95% 作为防幻觉硬门控。

本体 · ontology

四种节点，五类边。
足以装下一部小说。

Agent · Object · Event · Concept —— each shape carries its own truth conditions.

○ AGENT
行动者
有意图、有视角、能行动的实体
角色、叙述者、群体。携带信念、欲望、关系网；是 v0.4 Generative Agent 模拟的载体。
e.g. the narrator · John · the housekeeper
▢ OBJECT
物件
可被指称、被持有、被观察的实物
道具、场景元素、文档与信件。位置 / 所有权可随事件改变；进入 GLUCOSE 的 possession & location 维度。
e.g. the yellow wall-paper · the locked door
◇ EVENT
事件
realis-trigger，确实发生的
按 LitBank 标准严格定义 —— 不收假设、不收概括、不收否定、不收"角色相信但未发生"。
e.g. she pulled the paper down · he left at dawn
⬡ CONCEPT
概念
主题、意象、抽象指涉
由对象象征化或人物所信奉的非物质概念。作为 SYMBOLIZES 边的端点存在，是文学性的承载层。
e.g. confinement · sanity · domestic order

BRING YOUR OWN LLM

一个环境变量
切换任何 provider。

LoreGraph 兼容任何说 Anthropic native 或 OpenAI-compatible chat-completions 协议的 LLM。Anthropic 保留 prompt caching；OpenAI auto-cache 与 DeepSeek context cache 在 provider 暴露 cached_tokens 时自动累积。

一篇短篇（约 6000 词）：Anthropic 含 prompt cache 折扣 ~$0.20–1.00；DeepSeek / 通义 / Gemini Flash 通常便宜 3–10 倍；本地 Ollama 完全免费。

# Anthropic Claude (默认 · prompt caching)
LOREGRAPH_LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-...
# 切换到本地 Ollama —— 免 key、免费
LOREGRAPH_LLM_PROVIDER=ollama
LOREGRAPH_LLM_MODEL=llama3.2

ARCHITECTURE

单一 LLM 出口，
单一存储后端，
全链路证据接地。

所有 LLM 调用经过 llm/client.py —— token 计数、成本核算和 prompt caching 都依赖这条收口。

Pydantic ↔ SQLAlchemy 强制 lockstep；任何模型改动必须伴随一条 Alembic migration 与一个 round-trip 集成测试。
